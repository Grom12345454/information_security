# 1. Основная идея коррекции ошибок
*   **Проблема:** При передаче данных по реальным каналам связи (с помехами) могут возникать искажения — ошибки. Эти ошибки приводят к потере или искажению информации.
*   **Решение:** Ввести **избыточность** в передаваемые данные. Это означает добавление к исходной (информационной) части сообщения дополнительных битов (проверочных, контрольных), которые не несут полезной информации, но позволяют обнаружить или даже исправить ошибки на приемной стороне.
*   **Метод:** Использование **помехоустойчивых (корректирующих) кодов**. Эти коды строятся так, что разрешенные (разрешенные) кодовые комбинации отличаются друг от друга достаточно сильно, чтобы ошибки могли быть обнаружены или исправлены.
*   **Ключевой параметр:** **Минимальное кодовое расстояние `d₀`**. Это наименьшее расстояние Хэмминга между любыми двумя различными разрешенными кодовыми комбинациями в данном коде. Оно определяет корректирующую способность кода.
---
# 2. Кодовое расстояние и его свойства
*   **Определение:** Кодовое расстояние `d(A, B)` между двумя кодовыми комбинациями A и B — это число позиций (разрядов), в которых они различаются.
    *   *Пример:* d(1010, 1100) = 2 (различаются во 2-м и 3-м разрядах).
*   **Минимальное кодовое расстояние (`d₀`):** Наименьшее расстояние между любыми двумя различными разрешенными кодовыми комбинациями в данном коде. Это ключевой параметр, определяющий корректирующую способность кода.
*   **Геометрическая интерпретация:** Кодовые комбинации можно представить как вершины n-мерного куба (где n — длина кодового слова). Кодовое расстояние — это количество ребер, соединяющих две вершины (расстояние Хэмминга).
---
# 3. Связь минимального кодового расстояния с корректирующей способностью
*   **Обнаружение ошибок:**
    *   Код может обнаруживать все ошибки кратности до `r`, если `d₀ = r + 1`.
    *   *Пример:* Для обнаружения всех одиночных ошибок (`r=1`) необходимо `d₀ >= 2`. Если `d₀=2`, то любая одиночная ошибка переведет одно разрешенное слово в запрещенное, что будет обнаружено.
*   **Исправление ошибок:**
    *   Код может исправлять все ошибки кратности до `s`, если `d₀ >= 2s + 1`.
    *   *Пример:* Для исправления всех одиночных ошибок (`s=1`) необходимо `d₀ >= 3`. Это означает, что шары радиуса 1 (все слова, отличающиеся от данного на 1 бит) вокруг разных разрешенных слов не пересекаются. Принятое слово, содержащее одну ошибку, окажется ближе всего к исходному разрешенному слову, и его можно исправить.
*   **Совместное обнаружение и исправление:**
    *   Код может одновременно исправлять ошибки кратности до `s` и обнаруживать ошибки кратности до `r` (при `r > s`), если `d₀ >= r + s + 1`.

# 4. Расчет числа проверочных разрядов
Для кодов с заданным минимальным расстоянием `d₀` существует связь между числом информационных разрядов `nи`, числом проверочных разрядов `nк` и общей длиной кодового слова `n = nи + nк`.
*   **Для кодов, исправляющих одиночную ошибку (`d₀ = 3`):**
    *   Необходимо выполнение условия: $2^{nк} >= n + 1$.
    *   Практические формулы для расчета `nк`:
        *   Если известна общая длина `n`: $nк = log₂(n + 1)$ (целая часть логарифма).
        *   Если известно число информационных разрядов `nи`: $nк = log₂(nи + 1) + log₂(nи + 1)$.

# 5. Примеры простых методов обнаружения ошибок
*   **Контроль по четности (Паритетный контроль):**
    *   Добавляется один проверочный бит, значение которого (0 или 1) выбирается так, чтобы сумма всех битов в кодовой комбинации (включая проверочный) была четной (или нечетной).
    *   Обнаруживает все ошибки нечетной кратности (например, одиночные, тройные), но не обнаруживает ошибки четной кратности.
*   **Контроль по модулю:**
    *   Вычисляется контрольная сумма (остаток от деления суммы значений информационных разрядов на некоторый модуль M). Этот остаток передается как проверочное число.
    *   На приемной стороне вычисляется контрольная сумма снова. Если она не совпадает с принятой, значит, произошла ошибка.
*   **Контроль по произведению:**
    *   Аналогичен контролю по сумме, но используется произведение значений разрядов (часто по модулю).

# 6. Обнаружение ошибок в телеграфных кодах
*   **Телеграфный код №2 (Бодо):** Является равномерным пятиразрядным кодом без избыточности. Он не имеет возможности обнаруживать ошибки, так как все 32 возможные комбинации используются для передачи символов. Любая ошибка приведет к приему другого, но допустимого символа.
*   **Телеграфный код №3:** Содержит избыточность. Использует 7-битовые комбинации, но только 32 из 128 возможных комбинаций являются разрешенными. Остальные 96 комбинаций — запрещенные. Любая ошибка, переводящая разрешенную комбинацию в запрещенную, будет обнаружена.
