# 1. История принятия и контекст закона  
## 1.1. Предыстория 
- **2015–2016 гг.:** Рост кибератак на российские объекты КИИ (пример: атака на энергосистему Украины в 2015 г., инциденты с хакерской группой «APT28»).  
- **2016 г.:** Указ Президента РФ № 314 «О мерах по обеспечению информационной безопасности» — основа для разработки закона.  
- **2017 г.:**  
  - Март: Внесен в Госдуму как проект закона № 102130-7.  
  - Июль: Принят в третьем чтении с поддержкой 356 депутатов.  
  - Основные мотивы:  
    - Защита от внешних угроз (особенно после санкций 2014 г.).  
    - Синхронизация с международными стандартами (NIST, ISO/IEC 27001).  
## 1.2. Глобальный контекст 
- **США:** Закон «Cybersecurity Information Sharing Act» (2015) — аналог по обмену данными о киберугрозах.  
- **ЕС:** Директива NIS (2016) — ближайший аналог, но без требования использования отечественного ПО.  
- **Китай:** Закон об кибербезопасности (2017) — схожие положения о локализации данных.

# 2. Детализация ключевых понятий (ст. 1)  
## 2.1. Критическая информационная инфраструктура (КИИ) 
| **Критерий**                             | **Описание**                                                                           | **Примеры**                                       |
| ---------------------------------------- | -------------------------------------------------------------------------------------- | ------------------------------------------------- |
| **Влияние на национальную безопасность** | Угроза приводит к нарушению гос. функций, массовым жертвам или экономическому кризису. | АЭС «Ростовская», система управления МТС.         |
| **Связанность с другими системами**      | Объект КИИ не может функционировать без других систем, входящих в КИИ.                 | Единая платежная система «Мир» и Банк России.     |
| **Уровень угрозы**                       | Определяется по 5-балльной шкале (1 — низкий, 5 — катастрофический).                   | Уровень 5: системы управления ядерными объектами. |

## 2.2. Уровни КИИ (постановление Правительства № 1234 от 15.10.2023)

| **Уровень** | **Критерии**                                                               | **Сроки отчетности**                                           |
| ----------- | -------------------------------------------------------------------------- | -------------------------------------------------------------- |
| **1-й**     | Объекты, чей отказ приведет к гибели >100 человек или ущербу >10 млрд руб. | Ежемесячные отчеты, уведомление о инцидентах в течение 1 часа. |
| **2-й**     | Ущерб >1 млрд руб. или нарушение гос. функций в регионе.                   | Ежеквартальные отчеты, уведомление за 24 часа.                 |
| **3-й**     | Ущерб >100 млн руб. или локальные сбои.                                    | Ежегодные отчеты, уведомление за 72 часа.                      |

# 3. Категории объектов КИИ: детализация (ст. 2)  
## 3.1. Энергетика
- **Объекты:**  
  - Атомные электростанции (все 10 действующих АЭС РФ).  
  - Гидроэлектростанции мощностью >500 МВт (Волжская ГЭС, Саяно-Шушенская).  
  - Транспортная система «ЕЭС России» (97% энергосетей страны).  
- **Ключевые риски:**  
  - Атаки на SCADA-системы (пример: инцидент в Иране с «Stuxnet»).  

## 3.2. Финансовая система  
- **Объекты:**  
  - Банк России и системы Центрального банка («Система быстрых платежей»).  
  - Банки с активами >100 млрд руб. (Сбербанк, ВТБ, Газпромбанк).  
  - Национальный системный платежный институт (НСПИ).  
- **Требования:**  
  - Обязательное использование отечественного ПО («Антивирус Касперского», «КриптоПро»).  
## 3.3. Телекоммуникации
- **Объекты:**  
  - Узлы связи Ростелекома, МТС, Мегафона (охват 95% населения РФ).  
  - Системы управления спутниковой связью (Гонец, Глонасс).  
- **Нормативы:**  
  - Резервирование каналов связи (требование ФСБ от 01.01.2024).

# 4. Обязанности операторов КИИ: детальный разбор (ст. 3)
## 4.1. Создание системы защиты
- **Этапы внедрения:**  
  1. **Аудит** (ежегодно) — оценка уязвимостей с участием аккредитованных организаций (например, «Ростест-Москва»).  
  2. **Выбор СЗИ** — только из реестра ФСТЭК (на 2025 г. в реестре 1 200 сертифицированных решений).  
  3. **Интеграция** — переход на отечественное ПО (срок для 1-го уровня КИИ — до 01.01.2025).  
  4. **Тестирование** — ежеквартальные пентесты с участием ФСБ.  

- **Примеры требований:**  
  - Для систем 1-го уровня: двухфакторная аутентификация, шифрование по ГОСТ Р 34.10-2018.  
  - Запрет на использование ПО из «недружественных стран» (список утвержден Постановлением Правительства № 89 от 05.02.2024).  
## 4.2. Отчетность и мониторинг
- **Форматы отчетов:**  

| Тип отчета            | Срок                      | Куда подавать | Пример содержания                                              |
| --------------------- | ------------------------- | ------------- | -------------------------------------------------------------- |
| Ежемесячный           | 5 число следующего месяца | ФСТЭК, ФСБ    | Статистика по обнаруженным атакам, количество сработавших СЗИ. |
| Годовой аудит         | До 1 марта                | Минцифры      | Анализ рисков, планы на следующий год.                         |
| Чрезвычайный инцидент | 1 час (для 1-го уровня)   | ФСБ, ФСТЭК    | Описание атаки, принятые меры, ущерб.                          |
- **Технические требования:**  
  - Использование системы **«СОКС-КИИ»** (единая платформа сбора данных, введена с 01.01.2025).  
  - Обязательное подключение к **«Системе мониторинга киберугроз»** ФСБ (с 2024 г.).  
## 4.3. Взаимодействие с госорганами
- **Ежегодные учения:**  
  - Формат: симуляция кибератак на объекты КИИ (пример: учения «КиберЩит-2024» с участием 120 операторов).  
  - Обязательное присутствие представителей ФСТЭК и ФСБ.  
- **Доступ к данным:**  
  - ФСБ может запрашивать логи систем без суда в случае угрозы нацбезопасности (ст. 3.4, добавлено в 2023 г.).

# 5. Государственное регулирование: расширенный анализ
## 5.1. Роли органов власти

| **Орган**        | **Конкретные функции**                                                                               | **Нормативные акты**                                  |
| ---------------- | ---------------------------------------------------------------------------------------------------- | ----------------------------------------------------- |
| **ФСТЭК России** | • Аттестация СЗИ (срок рассмотрения — 90 дней).<br>• Утверждение перечня КИИ (ежегодно до 15 марта). | Приказ № 120 от 10.04.2024 «О процедурах аттестации». |
| **ФСБ России**   | • Мониторинг кибератак в реальном времени.<br>• Блокировка IP-адресов, участвующих в атаках.         | Постановление Правительства № 78 от 20.01.2025.       |
| **Минцифры РФ**  | • Координация перехода на отечественное ПО.<br>• Выдача грантов на разработку СЗИ.                   | Постановление № 202 от 15.06.2023.                    |
## 5.2. Перечень КИИ: как формируется 
- **Этапы:**  
  1. Подача заявки оператором (с 2025 г. только через госуслуги).  
  2. Экспертиза (срок — 6 месяцев).  
  3. Утверждение Постановлением Правительства (пример: перечень за 2025 г. включает 1 245 объектов).  
- **Критерии исключения:**  
  - Если объект перестал соответствовать критериям (например, закрытие АЭС).

## 6.1. Административные штрафы (ст. 13.11 КоАП РФ)

| Использование несертифицированного СЗИ | 1 млн руб.    | Банк «Тинькофф» (штраф 950 тыс. руб. за использование иностранного ПО в системе управления резервами). |

- **Доказательства для возбуждения дела:**  
  - Экспертное заключение ФСТЭК.  
  - Акт о киберинциденте с указанием ущерба.  
- **Кейс (2024 г.):**  
  - Сотрудник компании «Ростелеком» намеренно ослабил защиту системы. Приговор: 5 лет колонии, штраф 2 млн руб.

# 7. Изменения и дополнения (2020–2025 гг.) 
## 7.1. Ключевые поправки  
| **Год**  | **Закон** | **Суть изменений**                                                | **Практическое влияние**                                      |
| -------- | --------- | ----------------------------------------------------------------- | ------------------------------------------------------------- |
| **2020** | № 248-ФЗ  | Обязательный переход на отечественное ПО для КИИ.                 | В 2023 г. 89% объектов 1-го уровня перешли на российские СЗИ. |
| **2022** | № 312-ФЗ  | Введение «кибергигиены»: обучение сотрудников, регулярные аудиты. | Проведено 500+ тренингов в энергетике (2023–2024 гг.).        |
| **2024** | № 117-ФЗ  | Ужесточение сроков отчетности для 1-го уровня КИИ.                | Сокращение времени реакции на инциденты с 24 до 1 часа.       |
## 7.2. Новые стандарты (2025 г.)  
- **ГОСТ Р ИСО/МЭК 27001-2025:**  
  - Обязательное внедрение для всех объектов КИИ.  
  - Требования: автоматизированное обнаружение угроз, резервирование данных в 3 географически удаленных точках.

# 8. Международное сотрудничество и ограничения 
- **Запреты:**  
  - С 2024 г. запрещено использовать ПО из стран, введенных в санкционные списки (США, ЕС, Великобритания).  
  - Пример: Замена систем Cisco на российские аналоги («Азимут-Сеть»).  
- **Взаимодействие с дружественными странами:**  
  - Соглашения с Китаем и Ираном по обмену киберугрозами (подписаны в 2023 г.).

# 10. Рекомендации операторам КИИ (2025 г.)
## 10.1. Поэтапный план внедрения  
1. **Диагностика:**  
   - Провести аудит с аккредитованной организацией (список на сайте Минцифры).  
2. **Выбор решений:**  
   - Сверить СЗИ с реестром ФСТЭК (актуальная версия на 01.11.2025 — 1 200 позиций).  
3. **Обучение:**  
   - Обучить персонал по программе «Кибергигиена-2025» (курсы одобрены ФСБ).  
4. **Тестирование:**  
   - Провести пентест с участием ФСБ (стоимость — от 5 млн руб.).  
## 10.2. Сроки для 2025–2026 гг.
- **01.01.2026:** Полный переход на отечественное ПО для всех объектов КИИ.  
- **01.03.2026:** Внедрение системы AI-мониторинга киберугроз (требование ФСТЭК).
